### What worked well

  at render time) is the right architecture тАФ clean single-responsibility chain
- Extracting SVG JS into a `r##"..."##` raw string variable kept `format!()` free of `{{`/`}}`

### Problems, Root Causes, and Fixes

**Bug 1: Tab text black in light mode**

- **Root cause (compound):** Two separate bugs interacting.
  1. A duplicate CSS block at old lines 3113тАУ3140 from a prior session defined `.arch-tab.active`
     (JS sets `.arch-tab-active` тАФ a different class). The old block had `color: #fff` but was
     dead code since the JS never sets `.active` without the hyphen. Meanwhile...
  2. The new correct block at lines 3202+ had `.arch-tab-active { color: #0d0f14 !important }`.
     `#0d0f14` is a near-black hex that looks black on light backgrounds.
  3. The duplicate block also contained wrong SVG class names (`-hooks`, `-mods`) that didn't
     match any SVG markup, silently wasting cascade budget.
- **Fix:** Delete duplicate block; change `#0d0f14` тЖТ `#fff`

**Bug 2: Config files getting wrong syntax highlighting**

- **Root cause:** `collect_config_files` was designed before syntax highlighting existed and
  only returned `(title, content)` pairs. `render_config_section` emitted all files as
  `<pre class="config">` with no `data-lang`. `applyTomlHighlight()` in DOMContentLoaded
  then applied TOML highlighting to ALL `pre.config` elements тАФ including insights.md
  (should be markdown) and processed.json (should be JSON).
- **Fix:** Add `lang: String` as the third element of the tuple. Infer at collection time
  from the file extension. Emit as `data-lang` attribute. Replace `applyTomlHighlight` with
  `applyConfigHighlights` dispatcher.

**Bug 3: SVG diagram not interactive**

- **Root cause:** All 19 SVG nodes were bare `<rect>`+`<text>` elements. No `<g>` wrapper
  meant click/hover events on text didn't hit a common parent; there was no `data-svgid`
  to look up node info; no `SVG_NODE_INFO` table; no `initSvgNodes` JS; and `arch-detail`
  was inside the `arch-tab-grid` div so it was `display:none` when the Flow tab was active.
- **Fix:** Wrap all nodes in `<g data-svgid="...">`. Create `SVG_NODE_INFO` lookup table
  with 19 entries. Write `initSvgNodes` with mouseoverтЖТdim-unrelated, mouseoutтЖТrestore,
  clickтЖТshow-detail+highlight, related-linkтЖТjump-to-node. Move `arch-detail` outside
  both tab panels.

**Bug 4 (compilation): Raw string delimiter clash**

- **Root cause:** The `svg_js` variable was initially `r#"..."#`. The JS content included
  `<a href="#">` which contains the two-character sequence `"#` тАФ identical to the raw
  string terminator. Rust closed the string at that point, leaving the remaining JS as
  code to parse, producing confusing errors like `prefix 'link' is unknown`.
- **Fix:** Use `r##"..."##` тАФ the terminator becomes `"##` which never appears in HTML/JS.
  **Rule of thumb:** use at least `r##` for any raw string that will contain HTML attributes.
